<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL params → fetch | `[slug]` and `[id]` route params are interpolated into api-gateway fetch URLs |
| node data → navigation | `targetRoadmapId` drives client-side navigation |
| BlockNote JSON → DOM | `Node.content` (BlockNote JSON) is rendered into the page |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | Path traversal via [slug]/[id] | mitigate | Params are used only as URL path segments to api-gateway, which 404s unknown IDs (Phase 2). No filesystem access from params |
| T-03-05 | Elevation | Open redirect via targetRoadmapId | mitigate | RoadmapGraphView navigates ONLY to internal `/roadmap/${id}` paths via next/router push — never to an arbitrary external URL |
| T-03-06 | Tampering | XSS via BlockNote JSON content | mitigate | Content rendered exclusively through BlockNoteView (escapes content); `dangerouslySetInnerHTML` is forbidden |
| T-03-07 | Information Disclosure | NEXT_PUBLIC_API_URL exposure | accept | `NEXT_PUBLIC_*` is deliberately public (api-gateway URL); no secrets present |
| T-03-SC | Tampering | npm installs (next, @blocknote/*) | accept | RESEARCH.md Package Legitimacy Audit approved next@16.2.9 and @blocknote/* @0.51.4 (official Vercel/TypeCellOS packages) — no blocking checkpoint required |
</threat_model>
