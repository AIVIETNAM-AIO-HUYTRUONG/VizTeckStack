<objective>
Build `apps/svc-roadmap` — the NestJS 10 gRPC microservice that implements the
full `RoadmapService` contract (GetRoadmaps, GetRoadmap, GetNode, CreateRoadmap,
UpdateRoadmap, DeleteRoadmap, UpsertGraph). It is the sole owner of PostgreSQL
access via the `@vizteck/db` Prisma singleton (DEC-004), and consumes message
types from `@vizteck/proto` (DEC-003). UpsertGraph executes atomically in a
Prisma `$transaction`. The service listens on `0.0.0.0:5001` (configurable via
`GRPC_PORT`).

This plan covers Task 4 of the implementation plan
(`docs/superpowers/plans/2026-06-18-vizteckstack-implementation.md`, Task 4,
Steps 1–11). Implements REQ-grpc-service.

Purpose: Establish the only data-owning service so the api-gateway (Plan 02-02)
has a gRPC server to call. Without this, no data flows.
Output: A runnable gRPC server with 2 passing unit tests.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | gRPC port 5001 bound to 0.0.0.0 | accept | Local dev binds 0.0.0.0; in production firewall to internal network only (research Security Domain). No code change in this phase. |
| T-02-02 | Tampering | UpsertGraph `JSON.parse(n.content)` from proto string into Prisma Json? field | mitigate | Content originates only from the trusted gateway; transaction is atomic so a malformed parse rolls back the whole graph. Wrap parse so failures abort the $transaction rather than leaving partial state. |
| T-02-03 | Tampering | UpsertGraph partial save leaving orphaned nodes/edges | mitigate | Entire delete+reinsert runs inside db.$transaction (edges deleted before nodes per FK order). REQ-grpc-service success criterion. |
| T-02-SC | Tampering | npm package installs | mitigate | All deps are official NestJS/Google-gRPC ecosystem packages — RESEARCH.md Package Legitimacy Audit verdict OK/Approved for every package. No [ASSUMED]/[SUS] packages; no blocking checkpoint required. |
