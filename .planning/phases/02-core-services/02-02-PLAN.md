<objective>
Build `apps/api-gateway` — the single public HTTP boundary (DEC-002). It exposes:
- `POST /graphql` — Apollo Server 4, code-first `autoSchemaFile: true` (REQ-graphql-api)
- `GET|POST|PUT|DELETE /api/*` REST endpoints (REQ-rest-api)
- `GET /api-docs` — Swagger UI with Bearer auth

All write operations are protected by `AdminGuard` (static Bearer token, DEC-005),
which must work in both HTTP and GraphQL execution contexts. The gateway holds NO
database access (DEC-004) — it forwards everything to svc-roadmap (Plan 02-01) over
gRPC via `RoadmapGrpcClient`, converting Observables to Promises with `firstValueFrom`.

This plan covers Task 5 of the implementation plan
(`docs/superpowers/plans/2026-06-18-vizteckstack-implementation.md`, Task 5,
Steps 1–14). Implements REQ-rest-api and REQ-graphql-api.

Purpose: Give frontends (Phases 3–4) a GraphQL + REST surface and enforce admin auth.
Output: A runnable HTTP gateway with 2 passing resolver unit tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/client -> api-gateway (HTTP) | Untrusted input crosses here; admin writes must present a valid Bearer token. |
| api-gateway -> svc-roadmap (gRPC) | Internal trusted forward; gateway is the auth enforcement point. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Elevation of Privilege | Admin mutations/routes (POST/PUT/DELETE, GQL mutations) | mitigate | @UseGuards(AdminGuard) on every write resolver and REST route; AdminGuard validates Authorization: Bearer == ADMIN_TOKEN (DEC-005). |
| T-02-05 | Information Disclosure | Admin token accepted in query string | mitigate | AdminGuard reads ONLY the Authorization header; never a query param (research Security Domain). |
| T-02-06 | Spoofing | AdminGuard fails to read headers in GraphQL context | mitigate | Guard branches on context.getType(): http -> switchToHttp, else GqlExecutionContext.create(context).getContext().req (Pattern 5). |
| T-02-07 | Denial of Service | @Client gRPC options evaluated before .env loaded -> wrong ROADMAP_SERVICE_URL | accept | Plan reads process.env directly with localhost:5001 fallback; .env must be present before NestFactory.create (Pitfall 4). Acceptable for local dev; documented in .env.example. |
| T-02-SC | Tampering | npm package installs (incl. ts-morph ^24) | mitigate | All deps are official NestJS/Apollo/Google-gRPC ecosystem; RESEARCH.md Package Legitimacy Audit verdict OK/Approved. ts-morph is the published @nestjs/graphql peer. No [ASSUMED]/[SUS]; no blocking checkpoint required. |
</threat_model>
