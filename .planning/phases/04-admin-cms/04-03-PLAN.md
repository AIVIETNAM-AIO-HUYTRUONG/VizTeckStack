<objective>
Build the lesson editor at `/roadmaps/[id]/nodes/[nodeId]` (REQ-admin-lesson-editor): a client-side BlockNote editor that loads a LESSON node's existing content, lets the admin write rich text, and saves it. Because no `PUT /api/nodes/:id` exists, saving must load the FULL roadmap graph, merge the updated content into the target node, and call UpsertGraph with the complete node+edge list.

Purpose: This is the final authoring surface completing the platform. The critical constraint is that lesson save goes through the full-replace UpsertGraph endpoint, not a (nonexistent) node-update endpoint — so the page must hold the entire graph to avoid wiping other nodes.
Output: A working lesson editor whose saved content appears on the public viewer.
</objective>

Keep the FULL `nodes` + `edges` lists in state (needed because UpsertGraph is full replace — Pitfall 1). Find the target node by `nodeId` within the loaded graph nodes.
    2. Dynamic import the editor (BlockNote is browser-only): `const LessonEditor = dynamic(() => import('@/components/LessonEditor').then(m => m.LessonEditor), { ssr: false, loading: () => <div>Loading editor…</div> })`.
    3. Layout: below header, single column `max-w-3xl mx-auto px-lg py-lg`. Breadcrumb (14px text-3): `← Roadmaps / <roadmap title> / <node title>` — segments are links (`Roadmaps` → `/roadmaps`; roadmap title → `/roadmaps/<id>?slug=<slug>`). Page heading = node title (20px semibold).
    4. Render `<LessonEditor initialContentJson={targetNode.content ?? ''} onSave={handleSave} />`.
    5. `handleSave(contentJson)`: build the UpsertGraph payload from the FULL loaded graph — map every node to NodeInput shape `{ id, type, title, positionX, positionY, targetRoadmapId, content }`, and for the node whose id === nodeId, replace `content` with the new `contentJson` (already a JSON string; service will `JSON.parse` it). Map edges to `{ sourceId, targetId, label }`. POST to `/api/roadmaps/' + id + '/graph'` via apiFetch. CRITICAL: include ALL nodes and ALL edges unchanged except the one content field — sending a partial list deletes the rest (Pitfall 1). On success resolve (LessonEditor shows `Saved`); on failure let the promise reject so LessonEditor shows the error banner.
    6. Guard against missing data: if the node is not found in the graph, render a simple not-found message rather than crashing.

    The `content` field sent to UpsertGraph must be the BlockNote JSON serialized as a STRING (LessonEditor already passes `JSON.stringify(editor.document)`); do not double-stringify.
