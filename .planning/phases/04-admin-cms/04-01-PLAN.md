<objective>
Establish the foundation for the admin CMS: migrate the Prisma schema to make node positions nullable (required so unplaced nodes can persist), update the API DTO and svc-roadmap to tolerate null positions, extend shared graph types, scaffold the new `apps/admin` Next.js app, and implement the `/login` (REQ-admin-login) and `/roadmaps` CRUD (REQ-admin-roadmap-crud) screens.

Purpose: Without the nullable-position migration the graph editor (04-02) cannot persist unplaced inventory nodes. Without the app shell, login, and CRUD, no admin workflow can run. This plan unblocks both downstream plans.
Output: A buildable `apps/admin` app on port 3002 with working login and roadmap CRUD, plus a migrated schema and updated graph types.
</objective>
