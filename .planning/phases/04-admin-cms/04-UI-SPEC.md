# Phase 4 — Admin CMS UI Design Contract

> Visual and interaction contract for apps/admin. Generated by gsd-ui-researcher.
> Consumed by gsd-planner, gsd-executor, and gsd-ui-auditor.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual token system) |
| Preset | not applicable |
| Component library | packages/ui (Button, Card, NodeBadge — no build step) |
| Icon library | none (use text labels; Heroicons may be added inline if needed) |
| Font | Space Grotesk 700 (display), Inter 400/600 (body), JetBrains Mono 700 (mono/badge) |

## Spacing Scale

Declared values (4px base unit — Tailwind-compatible):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, tight inline gaps |
| sm | 8px | Button padding (vertical), compact list row gaps |
| md | 16px | Button padding (horizontal), card padding, form field spacing |
| lg | 24px | Section padding, modal body padding |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks (page-level separation) |
| 3xl | 64px | Not used in Phase 4 |

Exceptions:
- Header height: 56px (h-14 — established by apps/web layout; do not change)
- Touch / click targets minimum: 44px height for all interactive rows (inventory list rows, roadmap list rows)
- Node inventory drag handle: 40px wide, full row height — satisfies touch target without adding explicit padding
- Canvas toolbar height: 48px (fixed, above React Flow canvas)
- Side panel width: 320px (fixed, slides in from right)

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 14px | 400 (regular) | 1.5 | Inter | Table cells, form labels, list rows, meta text |
| Label | 14px | 600 (semibold) | 1.4 | Inter | Button labels, form field labels, section headers |
| Heading | 20px | 600 (semibold) | 1.3 | Inter | Page headings (h1 on /roadmaps, modal titles) |
| Display | 18px | 700 (bold) | 1.2 | Space Grotesk | App name in header only ("VizTeckStack") |
| Mono/Badge | 9px | 700 (bold) | 1.0 | JetBrains Mono | NodeBadge labels (ROADMAP / LESSON) — uppercase, letter-spacing 0.05em |

Rationale: Exactly 4 sizes (9, 14, 18, 20) and 2 weights (400, 600; 700 is display/badge only — not a body weight). Matches established apps/web pattern.

## Screen-by-Screen Interaction Contract

### Screen 1: /login

**Layout:** Full-page centered column. Single card (bg-1, radius-md, shadow-sm) centered at 400px width, vertically centered in viewport.

**Elements:**
- App name: `VizTeckStack Admin` — Display font, 20px, text-1
- Subheading: `Enter your admin token to continue` — Body, 14px, text-2
- Input: `Admin token` — password type (`type="password"`), full width, 14px Inter, bg-2 background, border, radius-sm, focus ring indigo, 40px height
- Primary CTA button: `Sign In` — variant=primary (indigo), full width, 40px height
- Error state: inline below input, 14px, red-500: `Invalid token. Please try again.`
- Loading state: button shows `Signing in…` and is disabled (opacity-60, cursor-not-allowed) during the validation fetch

**Interaction:**
- Submit on Enter key or button click
- On 200: store token in `localStorage.admin_token`, redirect to `/roadmaps`
- On 401: show error, clear input value, re-focus input
- On network error: show `Could not reach server. Check your connection.` (red-500, inline)

### Screen 2: /roadmaps

**Layout:** Full-width page. Header (h-14, bg-1, border-b) with app name left + ThemeToggle + Logout button right. Main content: max-w-4xl mx-auto, padding lg (24px).

**Header right controls:**
- Logout button: variant=ghost, 14px, text-2 label `Log out` — clears localStorage, redirects to /login
- ThemeToggle: copied from apps/web/src/components/ThemeToggle.tsx unchanged

**Page heading:** `Roadmaps` — 20px, semibold, text-1, mb-md (16px)

**Toolbar row (between heading and table):**
- `+ New Roadmap` button — variant=primary (indigo), right-aligned in the row

**Table / list:**
- Full-width table, bg-1, border, radius-md, overflow-hidden
- Header row: 14px semibold, text-2, bg-bg-2, border-b, px-md py-sm
- Columns: Title | Slug | Actions
- Data row: 14px regular, text-1, 44px min-height, border-b (last row: no border-b), hover bg-bg-2
- Slug cell: JetBrains Mono 12px, text-3
- Actions cell: Edit button (ghost, text-2, `Edit`) + Delete button (ghost, red-500, `Delete`) — 8px gap between

**Empty state (no roadmaps):**
- Heading: `No roadmaps yet`
- Body: `Create your first roadmap to get started.`
- CTA button: `+ New Roadmap` — variant=primary, centered below body text

**Create/Edit Roadmap modal:**
- Dialog overlay: bg-black/50 backdrop, centers a 480px card (bg-1, radius-md, shadow-lg)
- Modal heading: `Create Roadmap` or `Edit Roadmap` — 20px semibold, text-1
- Fields:
  - Title: text input, required, label `Title`, full width
  - Slug: text input, label `Slug`, full width, JetBrains Mono 13px in input, auto-generated (debounced 300ms from title), editable
  - Slug helper text: `Auto-generated from title. You can edit before saving.` — 12px, text-3
- Footer: Cancel button (variant=secondary) + Save button (variant=primary, `Create` or `Save Changes`) — right-aligned, 8px gap

**Delete roadmap confirmation:**
- Modal, 400px
- Heading: `Delete roadmap?`
- Body: `This will permanently delete "[roadmap title]" and all its nodes and edges. This cannot be undone.` — 14px, text-2
- Footer: Cancel (variant=secondary) + Delete (bg red-500, white text, `Delete Roadmap`) — right-aligned

### Screen 3: /roadmaps/[id] — Graph Editor

**Layout:** Three-zone vertical split within the page (below header):

```
[ Canvas toolbar (48px fixed) ]
[ React Flow canvas (flex-1, fills remaining height) ]
[ Node inventory (fixed 220px height, scrollable) ]
```

Total page: 100vh. Header: 56px. Remaining: calc(100vh - 56px) split into toolbar + canvas + inventory.

**Canvas toolbar (bg-1, border-b, px-md, h-12):**
- Left: back link `← Roadmaps` — ghost button, 14px, text-2
- Left (after back link): roadmap title — 14px semibold, text-1
- Right group (8px gap): `Add Node` button (variant=secondary) + `Save Graph` button
  - `Save Graph` default state: variant=primary (indigo)
  - `Save Graph` unsaved state: bg=#F59E0B (amber-500), text=white — indicates local state differs from saved
  - `Save Graph` saving state: `Saving…` label, disabled, opacity-60

**Canvas area:**
- `<RoadmapGraph mode="edit" />` fills full zone
- Background: bg-bg-0 (via React Flow backgroundVariant="dots" or "lines" with --bg-2 dot color)
- Dark mode canvas rules: already defined in globals.css — carry forward verbatim

**Right side panel (CreateNode / EditNode):**
- Slides in from right, 320px wide, full canvas-zone height
- Background: bg-bg-1, border-l border-border
- Slide transition: 200ms ease-out (transform translateX)
- Panel header: 14px semibold, text-1 — `Create Node` or `Edit Node`, with × close button (ghost, right-aligned)
- Fields:
  - Title: text input, label `Title`, required
  - Type: select/dropdown, label `Type`, options: `ROADMAP` / `LESSON`, with NodeBadge preview next to selected value
- Footer: `Cancel` (variant=ghost, text-2) + `Create Node` or `Save Changes` (variant=primary) — bottom of panel, border-t, px-md py-sm

**Trigger rules:**
- Right-click on empty canvas area → panel opens with mode=Create, position stored as canvas coordinates
- Click existing node on canvas → panel opens with mode=Edit, pre-filled with node data
- `Add Node` button in toolbar → panel opens with mode=Create, no canvas position (node placed in inventory only)

**Node inventory (bottom zone):**
- bg-bg-1, border-t border-border
- Section header bar (bg-bg-2, border-b, px-md, 32px): `Node Inventory` — 12px semibold, text-2, uppercase, letter-spacing 0.08em; right-aligned: node count badge (12px, text-3)
- Scrollable list of all nodes (placed + unplaced)
- Each row (44px min-height, border-b, flex, items-center, px-sm, 8px gap):
  - Drag handle (⠿ or ≡ icon): 40px wide, cursor-grab, text-3, flex-shrink-0
  - NodeBadge (type)
  - Node title: 14px, text-1, flex-1, truncate
  - Status chip: `Placed` (bg-indigo-lt, text-indigo, 11px, mono) or `Unplaced` (bg-bg-2, text-3, 11px, mono)
  - Delete button: ghost, red-500, 14px, `Delete`, flex-shrink-0
- Hover row: bg-bg-2

**Unplace vs. delete distinction (visual):**
- Deleting a node from the canvas (ReactFlow delete key or right-click) calls "unplace" — node moves to inventory with status `Unplaced`; no confirmation needed
- Deleting a node from the inventory triggers confirmation modal (destructive, permanent)

**Inventory delete confirmation:**
- Modal, 400px
- Heading: `Delete node?`
- Body: `This will permanently delete "[node title]" and all connected edges. This cannot be undone.` — 14px, text-2
- Footer: Cancel (variant=secondary) + `Delete Node` (bg red-500, white text)

**Unsaved changes navigation guard:**
- Before route change (Next.js router.push, browser back, link click) when unsaved === true:
- Modal, 420px
- Heading: `Leave without saving?`
- Body: `Your graph changes haven't been saved. Leaving now will discard them.` — 14px, text-2
- Footer: `Stay` (variant=secondary) + `Leave anyway` (variant=ghost, red-500 text) — right-aligned

### Screen 4: /roadmaps/[id]/nodes/[nodeId] — Lesson Editor

**Layout:** Below header. Single-column, max-w-3xl mx-auto, px-lg py-lg.

**Breadcrumb (above heading, 14px, text-3):**
`← Roadmaps / [roadmap title] / [node title]`
Each segment is a link (text-indigo on hover, no underline by default).

**Page heading:** Node title — 20px semibold, text-1, mb-md

**Editor area:**
- BlockNote editor (editable=true), loaded client-side only (`dynamic(() => ..., { ssr: false })`)
- Container: bg-bg-1, border, radius-md, px-lg py-md, min-height 400px
- BlockNote uses default theme; dark mode: pass `theme="dark"` when html.dark is active (read from document.documentElement.classList)

**Toolbar (above editor):**
- Right-aligned: `Save Lesson` button (variant=primary, indigo)
- Saving state: `Saving…`, disabled, opacity-60
- Success state: button briefly shows `Saved` with indigo background, reverts to `Save Lesson` after 2 seconds

**Error state (save failure):**
- Inline banner below toolbar: bg-red-50, border border-red-200, radius-sm, px-md py-sm, 14px, red-700 text
- Copy: `Failed to save. Please try again.`

## Accessibility Contract

- All interactive elements must have a visible focus ring: `outline: 2px solid #4F46E5; outline-offset: 2px`
- All `<button>` elements that are icon-only must have `aria-label`
- Modal dialogs: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to heading; focus trapped inside while open
- `<select>` for node type dropdown: no custom styling beyond border/bg; native select element
- Color alone must not convey the unsaved state — amber tint on Save Graph button is supplementary; a visually hidden accessible label `(unsaved changes)` should be added via `aria-label` override when unsaved === true
- Drag handles in inventory list: `aria-label="Drag to place on canvas"`, `role="button"`, `tabIndex={0}`
- Confirm dialogs: focus moves to Cancel button on open (destructive-default-safe)
