# Implementation Plan: User Management

**Branch**: `feature/003-user-management` | **Date**: 2026-07-01 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/003-user-management/spec.md`

## Summary

Thay thế single static `ADMIN_TOKEN` bằng hệ thống multi-user auth với JWT + bcrypt. Thêm User entity (SUPER_ADMIN / EDITOR / VIEWER) với CRUD, invite-via-email, và role-based route protection tại cả API layer và admin UI. Core-First: `AuthService` + `UserService` trong `packages/core`, apps là thin wrappers.

## Technical Context

**Language/Version**: TypeScript 5.4 (tất cả packages)

**Primary Dependencies**:
- NestJS (api-gateway): `@nestjs/jwt`, `@nestjs/passport`, `passport-jwt`, `bcrypt`, `nodemailer`
- Next.js 15 (admin): middleware.ts cho route protection
- Prisma (packages/db): schema thêm `User`, `AuditLog` models + enums

**Storage**: PostgreSQL via Prisma — migration required (new tables)

**Testing**: Jest + ts-jest (api-gateway), Vitest (packages/core, apps/admin)

**Target Platform**: Node.js server (api-gateway) + Next.js browser (admin)

**Performance Goals**: Login response < 3s (SC-001); role enforcement lag < 60s (SC-004)

**Constraints**:
- JWT expiry: 24h (per spec FR-002)
- Rate limit: 5 failed logins → 15min cooldown (per spec FR-015)
- < 100 users — no pagination required in v1

**Scale/Scope**: 3 new API modules (auth, users, email), 1 DB migration, 2 new admin pages

## Key Technical Notes

- **JWT storage**: Giữ key `admin_token` trong localStorage — `apiFetch` không cần thay đổi (backward compat)
- **AdminGuard deprecation**: Sau khi `JwtAuthGuard` hoạt động, remove `admin.guard.ts` và `ADMIN_TOKEN` env var
- **Login page layout**: Giống Graph Editor — standalone, KHÔNG dùng `AdminLayout`
- **`/api/auth/me`** là endpoint đầu tiên admin gọi sau page load để validate session
- **Password minimum**: 8 chars (không enforce complexity rules trong v1 — YAGNI)
- **Invite link format**: `http://localhost:3002/set-password?token=<inviteToken>` — cần thêm page `/set-password`
